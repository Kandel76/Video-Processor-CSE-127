## ADC Overview

This module implements a 4-bit Successive Approximation Register (SAR) Analog-to-Digital Converter (ADC) controller. The design assumes that one digital output code is generated for each conversion.

The ADC compares a sampled photodiode voltage (Vin) against a reference voltage generated by a Digital-to-Analog Converter (DAC). The comparator determines if the sampled input is greater than or less than the DAC output at each bit decision step.

The ADC module resolves the input voltage one bit at a time, and starts from the Most Significant Bit and ends at the Least Significant Bit.

**ADC Method**

In this design:
- `vinp` is the sampled photodiode voltage.
- `vinm` is the DAC-generated comparison voltage.
- `cmp_o` is the comparator output.
- `adc_dac_o` drives the DAC with the current trial code.
- `adc_o` is the final 4-bit digital output.

Conversion Principle: The SAR algorithm performs a binary search over the input voltage range.

For the 4-bit ADC:

1. Initializes the conversion with `adc_code = 0000`.
2. Set the MSB (bit 3) to 1 and send the trial code to the DAC.
3. The DAC generates a comparison voltage (Vtest).
4. The comparator evaluates:
   - If Vin > Vtest, keep the bit as 1.
   - If Vin ≤ Vtest, clear the bit to 0.
5. Move to the next lower bit (bit 2) and repeat.
6. Continue until all 4 bits have been resolved.
7. Store the final result in `adc_o`.

This process completes after four comparison cycles.

**Finite State Machine (FSM)**

The ADC operation is controlled by a finite state machine with the following states:

IDLE:
The ADC waits for a `read_en` signal.  
All internal counters and codes are reset.

VIN_HOLD:
The photodiode voltage is held stable for several clock cycles before starting bit comparisons.  
This ensures that the sampled input does not change during conversion.

CODE_SEND:
The current trial code is sent to the DAC.  
The DAC is given time to settle to the new reference voltage before the comparator decision is evaluated.

COMPARE:
The comparator output (`cmp_o`) is sampled.  
If `cmp_o = 1`, the current bit remains set.  
If `cmp_o = 0`, the bit is cleared.  
The bit index is decremented to move to the next lower bit.

CODE_STORE:
After all 4 bits are resolved, the final digital code is stored in `adc_o`, and the FSM returns to IDLE.

## Comparator Overview

The comp_gf180.sp file simulates a clocked comparator implemented with the GF180MCU standard-cell logic gates. The comparator is intended for use inside a Successive Approximation Register (SAR) ADC, where the comparator determines whether the sampled photodiode voltage (Vin) is greater than or less than the CDAC output voltage (Vtest).

This simulation is intended for:

- Comparator functionality validation
- SAR ADC architecture testing
- Offset estimation
- UCSC Academic research and coursework

It is not intended for final silicon verification.

In this simulation:
- `vinp` is the sampled photodiode voltage.
- `vinm` is the CDAC comparison voltage.
- The comparator resolves the differential input and outputs `q` and `qb`.

**Offset Calculation Method:**

1. Detect clock falling edges.
2. Sample `q` and `qb` after a small settle delay.
3. Make the decision:

       decision = (q > qb)

4. Detect the first low to high (0 to 1) transition.
5. Estimate threshold voltage:

       V_threshold = first Vinp where decision becomes HIGH

6. Calculate offset:

       Offset (mV) = (V_threshold − Vinm) × 1000

**How to Run Comparator Sim**

Run:

    ngspice -b tb_comp.sp

Then run the Python script to generate simulation plots:

- Decision vs Vinp plot
- Estimated comparator offset (mV)
